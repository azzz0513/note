### Linux中的线程和进程是怎么查看的
#### 1. 使用 ps 命令（静态快照）
ps 是最常用的查看进程状态的命令。
- **查看进程：**
    - ps aux：查看系统中所有的进程（BSD语法格式）。
    - ps -ef：查看系统中所有的进程（标准语法格式）。
- **查看线程：**
    - `ps -T -p <PID>`：查看指定进程的线程。**-T** 参数会开启线程查看，输出中会多出一列 **SPID**（或 TID），代表线程ID。
    - ps -eLf：查看系统中所有线程。输出中会多出一列 **LWP**（Lightweight Process），代表线程ID。NLWP列代表该进程有多少个线程。
    - 示例：ps -T -p 1234 （查看PID为1234的进程下的所有线程）

#### 2. 使用 top 命令（动态实时监控）
top 命令用于实时查看系统的资源使用情况。
- **查看进程：**
    - 直接输入 top，默认显示的就是各个**进程**的资源占用情况。
- **查看线程：**
    - **方法一（启动时指定）：** 输入 top -H，直接进入线程视图模式；或者 `top -H -p <PID>` 只监控某个进程下的线程。
    - **方法二（运行中切换）：** 在 top 界面运行期间，按下键盘上的大写字母 **H**，可以在“进程视图”和“线程视图”之间来回切换。切换到线程视图时，Tasks 统计行会变成 Threads。

#### 3. 使用 htop 命令（更直观的交互式工具）
htop 是 top 的增强版，支持鼠标点击，色彩丰富（如果没有安装，可通过 yum install htop 或 apt install htop 安装）。
- **查看进程：** 直接输入 htop。
- **查看线程：**
    - 在 htop 界面中，按下大写字母 **H** 可以显示/隐藏用户线程，按下大写字母 **K** 可以显示/隐藏内核线程。
    - 按下 **F5**（或小写字母 t）可以开启**树状视图**（Tree View），能够非常清晰地看到父进程、子进程以及挂载在进程下的线程结构（线程通常以绿色显示）。

#### 4. 使用 pstree 命令（树状图显示）
pstree 以树状图的方式展现进程之间的派生关系。
- **查看进程：**
    - pstree：显示进程树。
    - pstree -p：显示进程树，并同时显示每个进程的PID。
- **查看线程：**
    - `pstree -p <PID>`：查看指定进程的树状结构，默认会将线程包含在内。
    - `pstree -t -p <PID>`：明确展开该进程下的所有线程（在某些版本中，如果同名线程被折叠，加上 -t 可以展开）。在花括号 {} 中的通常是线程。

#### 5. 查看 /proc 虚拟文件系统（最底层的数据来源）
Linux 的所有进程和线程信息都以文件的形式存储在 /proc 目录下。
- **查看进程：**
    - 进程的信息存储在 `/proc/<PID>/` 目录下。
    - 你可以通过 `cat /proc/<PID>/status` 来查看某个进程的详细状态，里面有一行 Threads: 显示了该进程拥有的线程数量。
- **查看线程：**
    - 进程下的具体线程信息存储在 **`/proc/<PID>/task/`** 目录下。
    - 输入 `ls /proc/<PID>/task/`，你会看到一堆以数字命名的文件夹，这些数字就是该进程下各个**线程的TID（线程ID）**。
