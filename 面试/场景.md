## 怎么在1000亿数据（格式为URL）总找到重复次数最多的一个
使用 Hash 分片（Hash Sharding），将海量 URL 按哈希值分发到多个小文件中，相同 URL 一定进入同一个分片。然后分别统计每个分片中的 URL 频次，求出每个分片的 Top1，最后再从所有分片的 Top1 中找全局最大值。

### Step1：Hash(url)%N
比如：N = 1000
对于每条URL：
```Go
idx = hash(url)%1000
```
然后写入：
```
file_0
file_1
...
file_999
```
这样相同URL一定会进入同一个文件

#### 如果单个分片还是太大怎么办？
使用多级hash
如：
```
第一轮：1000 分片
第二轮：每个再分 1000
```
小到能内存处理

### Step2：统计每个小文件
对每个分片文件：
```Go
map[string]int
```
统计：
```Go
count[rul]++
```
然后找到当前文件中重复次数最多的记录下来，最后得到1000个记录

### Step3：再求全局Top1
最后只需要比较：1000 个候选人
即可：全局出现次数最多的 URL
复杂度极低。
