## 死信队列
处理“无法被正常消费”的消息（如消费超时、消费失败且重试次数耗尽），将其转移到专门的”死信“队列，以便后续问题的排查。
- 为普通队列设置死信交换器（`x-dead-letter-exchange`）和死信路由键（`x-dead-letter-routing-key`）
- 当消息满足死信条件时，会被路由到死信队列

正常生产者：
1. 先声明普通交换器
2. 声明普通队列，通过额外参数设置死信交换器
3. 绑定普通队列到普通交换器
4. 发送消息到普通队列

```Go
package main  
  
import (  
    amqp "github.com/rabbitmq/amqp091-go"  
    "log"    
    "os"    
    "strings")  
  
func failOnError(err error, msg string) {  
    if err != nil {  
       log.Panicf("%s: %s", msg, err)  
    }  
}  
  
func bodyFrom(args []string) string {  
    var s string  
    if (len(args) < 2) || args[1] == "" {  
       s = "hello"  
    } else {  
       s = strings.Join(args[1:], " ")  
    }  
    return s  
}  
  
func main() {  
    conn, err := amqp.Dial("amqp://guest:guest@localhost:5672/")  
    failOnError(err, "Failed to connect to RabbitMQ")  
    defer conn.Close()  
  
    ch, err := conn.Channel()  
    failOnError(err, "Failed to open a channel")  
    defer ch.Close()  
  
    // 声明普通交换器  
    err = ch.ExchangeDeclare(  
       "normal_exchange", // name  
       "direct",          // type  
       true,              // durable  
       false,             // auto-deleted  
       false,             // internal  
       false,             // no-wait  
       nil,               // arguments  
    )  
    failOnError(err, "Failed to declare an exchange")  
  
    // 声明普通队列并设置死信交换器  
    args := amqp.Table{  
       "x-dead-letter-exchange":    "dlx_exchange",    // 死信交换器  
       "x-dead-letter-routing-key": "dlx_routing_key", // 死信路由键  
    }  
    _, err = ch.QueueDeclare(  
       "normal_queue", // name  
       true,           // durable  
       false,          // delete when unused  
       false,          // exclusive  
       false,          // no-wait  
       args,           // arguments  
    )  
    failOnError(err, "Failed to declare a queue")  
  
    // 绑定普通队列到普通交换器  
    err = ch.QueueBind(  
       "normal_queue",       // queue name  
       "normal_routing_key", // routing key  
       "normal_exchange",    // exchange  
       false,  
       nil,  
    )  
    failOnError(err, "Failed to bind a queue")  
  
    // 发送消息到普通队列  
    body := bodyFrom(os.Args)  
    err = ch.Publish(  
       "normal_exchange",    // exchange  
       "normal_routing_key", // routing key  
       false,                // mandatory  
       false,                // immediate  
       amqp.Publishing{  
          ContentType: "text/plain",  
          Body:        []byte(body),  
       })  
    failOnError(err, "Failed to publish a message")  
    log.Printf(" [x] Sent %s", body)  
}
```

```Go
       false,          // auto-deleted  
       false,          // internal  
       false,          // no-wait  
       nil,            // arguments  
    )  
    failOnError(err, "Failed to declare an exchange")  
  
    // 声明死信队列  
    q, err := ch.QueueDeclare(  
       "dlx_queue", // name  
       true,        // durable  
       false,       // delete when unused  
       false,       // exclusive  
       false,       // no-wait  
       nil,         // arguments

       "dlx_queue",  
       "dlx_routing_key",  
       "dlx_exchange",  
       false,  
       nil,  
    )  
    failOnError(err, "Failed to bind a queue")  
  
    // 消费死信队列中的消息  
    msgs, err := ch.Consume(  
       q.Name, // queue)  
       "",     // consumer  
       true,   // auto-ack  
       false,  // exclusive  
       false,  // no-local  
       false,  // no-wait  
       nil,    // args  
    )  
    failOnError(err, "Failed to register a consumer")  
  
    forever := make(chan bool)  
    go func() {  
       for d := range msgs {  
          log.Printf("Received a message: %s", d.Body)  
          // 拒绝消息并不重新入队，触发死信机制  
          d.Reject(false)  
          log.Printf("Rejected: %s", d.Body)  
       }  
    }()  
    log.Printf(" [*] Waiting for messages. To exit press CTRL+C")  
    <-forever  
}
```

## 延迟队列
消息发送后不会立即被消费，而是延迟指定时间后才被处理。

```Go
args := amqp.Table{  
    "x-dead-letter-exchange":    "dlx_exchange",    // 死信交换器  
    "x-dead-letter-routing-key": "dlx_routing_key", // 死信路由键  
    "x-message-ttl":             5000,              // 5秒后消息超时过期
}  
_, err = ch.QueueDeclare(  
    "normal_queue", // name  
    true,           // durable  
    false,          // delete when unused  
    false,          // exclusive  
    false,          // no-wait  
    args,           // arguments  
)
```

## 优先级队列
在队列声明时通过`x-max-priority`参数设置队列支持的最大优先级
发送带优先级的消息
在`amqp.Publishing`中设置`Priority`字段指定消息的优先级，消费者会优先收到优先级较高的消息

```Go
args := amqp.Table{  
    "x-max-priority": 10, // 设置最大优先级
}  
_, err = ch.QueueDeclare(  
    "normal_queue", // name  
    true,           // durable  
    false,          // delete when unused  
    false,          // exclusive  
    false,          // no-wait  
    args,           // arguments  
)
```
```Go
err = ch.Publish(

    "normal_routing_key", // routing key

    })
```

## 流量控制
生产者发送消息，如果队列消息已经积压了，则生产者慢点发
消费者按照自己的能力消费消息
生产者通过`ch.QueueInspect(q.Name)`获取队列状态
消费者通过`ch.Qos()`设置消费的能力
```Go
package main  
  
import (  
    amqp "github.com/rabbitmq/amqp091-go"  
    "log"    
    "os"    
    "strings"
    )  
  
func failOnError(err error, msg string) {  
    if err != nil {  
       log.Panicf("%s: %s", msg, err)  
    }  
}  
  
func bodyFrom(args []string) string {  
    var s string  
    if (len(args) < 2) || args[1] == "" {  
       s = "hello"  
    } else {  
       s = strings.Join(args[1:], " ")  
    }  
    return s  
}  
  
func main() {  
    conn, err := amqp.Dial("amqp://guest:guest@localhost:5672/")  
    failOnError(err, "Failed to connect to RabbitMQ")  
    defer conn.Close()  
  
    ch, err := conn.Channel()  
    failOnError(err, "Failed to open a channel")  
    defer ch.Close()  
 
	// 声明队列  
    q, err = ch.QueueDeclare(  
       "normal_queue", // name  
       true,           // durable  
       false,          // delete when unused  
       false,          // exclusive  
       false,          // no-wait  
       args,           // arguments  
    )  
    failOnError(err, "Failed to declare a queue")   
  
	maxQueueLength := 100 // 队列最大长度阈值
  
    // 发送消息到普通队列  
    for i := 1; i <= 1000; i++ {
	    // 获取队列状态
	    queueStatus, err := ch.QueueInspect(q.Name)
	    failOnError(err, "Failed to inspect queue")
	    
	    // 如果队列长度超过阈值，则等待
	    if queueStatus.Message >= maxQueueLength {
		    log.Printf("队列长度 %d 超过阈值 %d，等待...", queueStatus.Message, maxQueueLength)
		    time.Sleep(2*time.Second)
		    i-- // 重试当前消息
		    continue
	    }    
	    body := bodyFrom(os.Args)  
		err = ch.Publish(  
		    "normal_exchange",    // exchange  
		    "normal_routing_key", // routing key  
	        false,                // mandatory  
	        false,                // immediate  
		    amqp.Publishing{  
		        ContentType: "text/plain",  
		        Body:        []byte(body),  
		    })  
	    failOnError(err, "Failed to publish a message") 
	    log.Printf(" [x] Sent %s", body) 
    }
}
```

## 消息过期
在延迟队列，我们直到队列可以设置整体的消息过期时间。
其实消息本身也可以设置过期时间
核心就是：
```Go
amqp.Publishing{
	Body: []byte(msg.Body)
	Expiration: fmt.Sprintf("%d", msg.ttl)  //消息过期时间（毫秒）
}
```

## 持久化
需要设置队列和消息的持久化
声明持久化队列：
```Go
// 声明一个持久化的队列
q, err := ch.QueueDeclare(
	"normal_queue", // 队列名称
	true, // 是否持久化
	false, // 是否自动删除
	false, // 是否独占
	false, // 是否等待服务器响应
	nil, // 额外参数
)
```

声明持久化交换器：
```Go
// 声明持久化的交换器
err = ch.ExchangeDeclare(  
   "normal_exchange", // name  
   "direct",       // type  
   true,           // durable  
   false,          // auto-deleted  
   false,          // internal  
   false,          // no-wait  
   nil,            // arguments  
)
```

发送持久化消息：
```Go
// 要发送的消息
body := "Hello"
// 发布消息到队列，设置为持久化
err = ch.Publish(
	"", // 交换器
	q.Name, // 路由键（队列名称）
	false, // 是否强制
	false, // 是否立即
	amqp.Publishing{
		ContentType: "text/plain",
		DeliveryMode: amqp.Persistent,
		Body: []byte(body),
	})
```

## 参数详解
### 交换器的参数
```Go
func (ch *Channel) ExchangeDeclare(name, kind string, durable, autoDelete, internal, noWait bool, args Table) error
```
- `name`：交换器的名称（`""`为直接交换器，就是默认交换器）
- `kind`：交换器的类型（如`fanout`、`direct`、`topic`、`headers`）
- `durable`：持久化
	- `true`表示交换器是持久化的，在broker重启后交换器依然存在
	- `false`表示交换器是非持久化的，在broker重启后会被删除
	- 这只保证交换器本身的持久化，并不持久化发送到给交换器的消息
- `autoDelete`：自动删除
	- `true`表示在所有队列都解除与此交换器的绑定后，该交换器会自动删除
	- `false`表示即使当前没有队列绑定它，交换器也不会被自动删除，需要手动删除
- `internal`：是否将交换器设置为内部交换器
	- `true`：此交换器为内部交换器。客户端不能直接向它发布消息。
	- `false`：此交换器为普通交换器，客户端可以向其发布消息
- `noWait`：是否异步声明，不等待服务器的确认响应
	- `true`：方法调用后立即返回，不等待服务器返回声明是否成功的响应。如果参数错误导致声明失败，可能会在后续操作中才返回错误，甚至导致通道关闭。
	- `false`: 方法会阻塞并等待服务器的确认响应，确保交换机已成功声明后再返回。
	-  除非你非常追求性能且能确保参数正确，否则建议设置为 `false` 以保证可靠性。
- `args`：`amqp.Table`类型
	- 用于设置额外的可选参数。这是一个键值对映射（`map[string]interface{}`）


### 队列的参数
```Go
func (ch *Channel) QueueDeclare(name string, durable, autoDelete, exclusive, noWait bool, args Table) (Queue, error)
```
- `name`：队列名称
	- 这是你要声明的队列的名称
	- 如果为空字符串，RabbitMQ会自动生成一个随机名称
- `durable`：持久化
	- `true`表示队列是持久化的，在broker重启后队列依然存在
	- `false`表示队列是非持久化的，在broker重启后队列会被删除
	- 这只保证队列本身的持久化，不保证消息的持久化
- `autoDelete`：自动删除
	- `true`：表示当最后一个消费者断开连接后，队列会自动删除
	- `false`：表示队列不会自动删除，即使没有消费者连接
- `exclusive`：排他性
	- `true`表示队列是排他的，只能被当前连接使用，连接关闭后队列会被删除
	- false表示队列可以被多个连接共享
- `noWait`：不等待
	- `true`表示不需要等待服务器的确认，会立即返回
	- `false`表示会等待服务器的确认后才返回
- `args`：额外参数
	- 用于指定队列的额外属性，如消息TTL、最大长度等
	- `nil`表示使用默认设置

声明队列的时候，相同名称的队列，如果重复声明，必须保证所有配置一样。

设置队列的消息长度
```Go
args := amqp.Table{
	"x-max-length": 100, // 最大消息数100
	"x-overflow": "reject-publish", // drop-head丢弃旧消息，reject-publish超过限制时拒绝新消息
}
```


### 消息的参数
```Go
func (ch *Channel) Publish(exchange, key string, mandatory, immediate bool, msg Publishing) error
```
- `exchange`：交换器的名称
- `key`：路由键，如果`exchange = ""`，则路由键=队列名
- `mandatory`：为`true`时，如果消息无法被路由到任何队列（没有匹配的绑定规则），RabbitMQ会键消息返回给生产者
- `immediate`：如果消息路由到的队列没有消费者，则消息会被返回给生产者
- `msg`：消息内容

`Publishing`字段详解：
```Go
type Publishing struct {  
	// 应用程序或交换器特定的字段，headers类型交换器会检查该字段
	// 用于存储自定义键值对元数据，可用于headers交换器的路由规则匹配   
    Headers Table  
  
    // 消息属性
    ContentType     string // MIME内容类型（如"text/plain"，"application/json"） 
    ContentEncoding string // MIME内容编码（如"gzip"，"base64"）
    DeliveryMode    uint8  // 投递模式：0或1为非持久化，2为持久化（需配合持久化队列）
    Priority        uint8  // 消息优先级（0~9，需队列支持x-max-priority参数）
    CorrelationId   string // 关联标识符，用于匹配请求与响应（常用于RPC）
    ReplyTo         string // 回复地址，指定接收响应的回调队列（常用于RPC） 
    MessageId  string    // 消息唯一标识符（业务层面，用于追踪或去重）
    Timestamp  time.Time // 消息创建的时间戳
    Type       string    // 消息类型名称（自定义）
    UserId     string    // 发送消息的用户ID  
    AppId      string    // 发送消息的应用长须ID  
  
    // 消息的实际业务数据负载    
    Body []byte  
}
```

### 消费者的参数
```Go
func (ch *Channel) Consume(queue, consumer string, autoAck, exclusive, noLocal, noWait bool, args Table) (<-chan Delivery, error)
```
- `queue`：字符串类型，指定要消费的队列名称
- `consumer`：字符串类型，消费者的标识名称。可以自定义命名，用于在RabbitMQ管理界面中识别不同的消费者
- `autoAck`：布尔类型，是否自动确认消息
	- `true`：消息被消费者接收后会自动确认（从队列中移除），无论业务处理是否成功
	- `false`：需要手动调用`Delivery.Ack()`方法确认消息，确保业务处理完成后再删除消息
- `exclusive`：布尔类型，是否设置为排他性消费者
	- `true`：该消费者是队列的唯一消费者，其他消费者无法消费该队列
	- `false`：允许多个消费者同时消费该队列（消息会轮询分配给不同消费者）
- `noLocal`：布尔类型，是否禁止接收本连接发布的消息，需要借助direct交换器
	- `true`：消费者不会受到由当前连接发布得到消息
	- `false`：可以接收当前连接发布的消息
- `noWait`：布尔类型，是否非阻塞等待服务器响应
	- `true`：方法会立即返回，不等待服务器确认
	- `false`：等待服务器确认后再返回（推荐使用）
- `args`：`Table`类型，额外的参数配置。通常用于传递RabbitMQ扩展属性（如消息优先级，过期时间等），一般情况下可以传`nil`

`msg.Ack(false)`表示收到此消息，此消息会从队列中被删除
如果不执行`Ack`也不执行`Reject`，那么下次运行消费者，此消息会被传递给消费者
`msg.Reject(true)`，表示拒绝消息，并且让消息重新入队
`msg.Reject(false)`，表示拒绝消息，此消息会被从队列中删除
如果消费者接收消息的处理任务存在失败的风险，那么建议手动确认消息
