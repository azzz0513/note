数据库连接池（database connection pool）是一种用于管理和复用数据库连接的技术。它的核心思想是：在应用程序启动时预先创建一定数量的数据库连接，并将这些连接保存在一个“池“中，当需要访问数据库时，从池中获取一个空闲连接，使用完毕后不是关闭连接，而是将其归还到池中，供后续请求复用。

### 为什么需要连接池
建立和关闭数据库连接是一个开销较大的操作：
- TCP连接的三次握手
- 数据库的身份校验
- 初始化会话上下文

如果每次数据库操作都新建和关闭连接，会导致：
- 性能下降
- 资源浪费
- 系统不稳定

### 连接池的主要作用
- 提高性能：复用已有连接，避免频繁创建/销毁连接的开销
- 控制资源使用：限制最大连接数，防止数据库因连接过多而崩溃
- 提高系统稳定性与可伸缩性：如连接超时回收、空闲连接检测、自动重连、连接健康检查等

### 关键参数

| 参数                    | 默认值      | 说明                            |
| --------------------- | -------- | ----------------------------- |
| `max_connection`      | 151      | 最大并发连接数。超过此值，新连接会被拒绝          |
| `wait_timeout`        | 28800（s） | 非交互式连接空闲超时时间（如gorm连接），超时后自动断开 |
| `interactive_timeout` | 28800（s） | 交互式连接空闲超时时间（如mysql命令行）        |
| `max_connect_errors`  | 100      | 主机连续连接失败次数超过此值，会被临时屏蔽         |
| `connect_timeout`     | 10（s）    | 建立连接的超时时间                     |
| `thread_cache_size`   | 9        | 线程缓存大小，减小创建新线程的开销             |
