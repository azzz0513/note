# 负责模块及具体工作
## 科研版
### 功能
1. 康养body服务：
	1. 用户数据查询（AttributeUserData）：查询科研版授权用户（根据授权手表）获取指定时间范围内指定数据
	2. 用户数据列表下拉筛选（AttributeUserDataFilterList）：获取用户数据查询可筛选的数据项、Sn项、数据类型项
	3. 用户数据导出（AttributeUserDataExport）：将用户数据导出为excel格式文件
2. 稳态user服务：
	1. 扫码授权科研版：
		1. 微信公众号登录（Login）：新增bindScientific
		2. 微信绑定用户服务（WechatBind）：新增bindScientific
		3. 微信公众号登录注册用户（WechatRegister）：新增bindScientific
3. 康养user服务：
	1. `user_property.go`新增字段：

- 数据源表：用于配置数据源与相关机构的授权关系
```SQL
CREATE TABLE `tb_service_data_human_attribute_source`  (
  `id` varchar(255) CHARACTER SET utf8mb3 COLLATE utf8mb3_bin NOT NULL COMMENT '唯一标识uuid',
  `is_auth` varchar(64) CHARACTER SET utf8mb3 COLLATE utf8mb3_bin NOT NULL COMMENT '数据源是否授权（\"0\"：未授权 \"1\"：授权对应机构）',

  `client_id` varchar(64) CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci NOT NULL COMMENT '数据源客户端',
  `hospital_id` varchar(64) CHARACTER SET utf8mb4 COLLATE utf8mb4_0900_ai_ci NOT NULL COMMENT '数据源机构',
  `create_time` datetime(6) NOT NULL COMMENT '数据创建时间',
  `creator` varchar(255) CHARACTER SET utf8mb3 COLLATE utf8mb3_bin NOT NULL COMMENT '数据创建者',
  `update_time` datetime(6) NOT NULL COMMENT '数据更新时间',
  `updator` varchar(255) NOT NULL COMMENT '数据更新者',
  `source` varchar(255) CHARACTER SET utf8mb3 COLLATE utf8mb3_bin NOT NULL COMMENT '数据源sn码',
  PRIMARY KEY (`id`)
) ENGINE = InnoDB CHARACTER SET = utf8mb3 COLLATE = utf8mb3_bin COMMENT = '数据源表' ROW_FORMAT = DYNAMIC;
```
