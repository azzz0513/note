## 基本数据类型
- 字符串String
- 列表List
- 集合Set
- 有序集合SortedSet
- 哈希Hash

### String
Redis中的数据是以键值对的形式存储的，所以我们需要指定一个键和一个值
Redis中默认是以字符串形式存储数据，并且是二进制安全的
- 设置键值对（键是大小写敏感的）：`SET key value`
- 获取键的值：`GET key`
- 删除一个键：`DEL key`
- 判断一个键是否存在：`EXISTS key`
- 查看数据库中有哪些键：`KEYS pattern`
	- pattern用于匹配字符串参数
- 将数据库中所有的键删除：`FLUSHALL`
- 查看一个键的过期时间：`TTL key`
- 设置过期时间：`EXPIRE key time`
- 设置一个带有过期时间的键值对：`SETEX key time value`
- 当键不存在时设置键值对，如果存在则不进行任何动作：`SETNX key value`


### List
一般用于存储和操作一组有顺序的数据，类似于数组
- 将元素添加到列表的头部：`LPUSH key value`
	- 也可一次添加多个元素，按照顺序依次入队
- 将元素添加到列表的尾部：`RPUSH key value`
	- 也可一次添加多个元素，按照顺序依次入队
- 获取列表内容：`LRANGE key start_pos end_pos`
	- end_pos位置为-1，表示最后一个元素
- 从列表头部删除元素：`LPOP key`
	- 也可一次删除多个元素，只需在最后添加删除元素个数即可
- 从列表尾部删除元素：`RPOP key`
	- 也可一次删除多个元素，只需在最后添加删除元素个数即可
- 查看列表的长度：`LLEN key`
- 删除指定范围以外的元素：`LTRIM key start_pos end_pos`


### Set
是一种无序结构，与列表的区别为列表中可以有重复的元素，而集合中不能有重复的元素
- 添加元素到集合：`SADD key member`
	- 值可以有一个或多个
- 查看集合中的元素：`SMEMBERS key`
- 判断一个元素是否在集合中：`SISMEMBER key member`
- 删除集合中的元素：`SREM key member`
- 集合运算

### SortedSet（ZSet）
有序集合的每个元素都会关联一个浮点类型的分数，然后根据分数对集合中的元素进行从小到大的排序。有序集合的成员是唯一的，但是分数可以是重复的，当分数相同时，按照字典序进行排序。
- 添加元素到有序集合：`ZADD key score member`
- 查看有序集合中的元素：`ZRANGE key start_pos end_pos`
	- 默认只输出成员，不输出分数，可在最后加上`WITHSCORES`来指定输出分数
- 查看指定成员的分数：`ZSCORE key member`
- 查看指定成员的排名（从小到大，返回的为索引值）：`ZRANK key member`
- 查看指定成员的排名（从大到小）：`ZREVRANK key member`
- 删除指定成员：`ZREM key member`

### Hash
是一个字符类型的字段和值的映射表，简单理解就是一个键值对的集合，适合用来存储对象。
- 向哈希中添加键值对：`HSET key field value`
- 获取哈希中的指定键值：`HGET key field`
- 获取整个哈希中的所有键值对：`HGETALL key`
- 删除指定键值对：`HDEL key field`
- 判断某个键值对是否存在：`HEXISTS key field`
- 获取哈希中的所有键：`HKEYS key`
- 获取哈希中所有键值对的数量：`HLEN key`

## 高级数据类型
- 消息队列Stream
- 地理空间Geospatial
- HyperLogLog
- 位图Bitmap
- 位域Bitfield

### Stream
- 向消息队列添加消息：`XADD key id messgae`
	- id处为`*`，则Redis会自动保证id为递增的，如果为自己手动指定id，需要自己保证id的递增
	- id：为`X-X`的格式，第一个X为时间戳，第二个X表示序列号
- 查看消息队列中消息的数量：`XLEN key`
- 查看消息队列中的消息的详细内容：`XRANGE key start_pos end_pos`
	- 可以使用`XRANGE key - +`来显示所有消息
- 删除消息：`XDEL key id`
- 删除指定范围以外的消息：`XTRIM key `
- 读取消息：`XREAD COUNT count BLOCK time STREAMS key id`
	- COUNT：指定一次读取消息的数量
	- BLOCK：指定没有消息时的阻塞时间（millisecond）
	- STREAMS：指定获取消息的消息队列
- 创建消费者组：`XGROUP CREATE key group id`
- 查看消费者组的信息：`XINFO GROUPS key`
- 添加消费者到指定组：`XGROUP CREATECONSUMER key group consumer`
- 读取消息：`XREADGROUP GROUP group consumer COUNT count BLOCK time STREAMS key id`

#### 发布订阅模式
- 将消息发送到指定的频道：`PUBLISH channel message`
- 订阅指定的频道，就可以接收到这个频道的消息：`SUBSCRIBE channel`

### Geospatial
提供一种存储地理位置信息的数据结构，同时支持对各种地理位置进行各种计算操作。
- 添加地理位置信息：`GEOADD key longitude latitude member`
- 获取某个地理位置的经纬度：`GEOPOS key member`
	- 第一个返回值为经度，第二个返回值为纬度
- 计算两个地理位置之间的距离：`GEODIST key member1 member2`
- 搜索指定范围内的成员并返回：`GEOSEARCH key FROMMEMBER member | FROMLONLAT longitude latitude BYRADIUS radius | BYBOX width height`

### HyperLogLog
是一种用于做基数统计的算法（如果集合中的每个元素都是唯一且不重复的，那么这个集合的基数就是集合中元素的个数）。适合用于一些对精确度要求不高，但是数据量非常的统计工作。
- 添加元素：`PFADD key element`
- 查看基数：`PFCOUNT key`
- 将一个或多个HyperLogLog值合并为一个键：`PFMERGE result_key key1 key2 ...`


### Bitmap
位图是字符串类型的扩展，可以使用一个String类型来模拟一个BItmap，数组的下标就是偏移量，值只有0和1，也支持一些位运算。
- 设置偏移量的值：`SETBIT key offset value`
- 获取某个偏移量的值：`GETBIT key offset`
- 可以直接使用String类型命令：
	- 设置偏移量的值（可以用十六进制数来表示二进制数）：例：让前4位为1，后4位为0：`SET test "\xF0"`
- 统计某一个key的值里面有多少个位为1：`BITCOUNT key`
- 获取某个key里面第一个出现的0或1的位置：`BITPOS key bit`

### Bitfield
位域能够将很多小的整数存储到一个较大的位图中，这样就可以更高效地使用内存
- 对String执行任意位域整数运算：`BITFIELD key [GET encoding offset | [OVERFLOW WRAP | SAT | FAIL] SET encoding offset value | INCRBY encoding offset increment [GET encoding offset | [OVERFLOW WRAP | SAT | FAIL] SET encoding offset value | INCRBY encoding offset increment ...]]`
	- encodeing：指定数据格式
- 查看信息：`GET key`
