## 中间代码生成（Intermediate Code Generation）
代码生成的任务是为目标计算机生成可执行的代码，该代码忠实的表示源代码的语义
代码生成通常分为几个步骤：
- 中间代码生成
- 生成某种形式的汇编代码
- 优化（Optimization）：提升目标代码的速度和大小

中间代码生成：
- 代码生成的中间代码
- 基本代码生成技术
- 控制语句和逻辑表达式的代码生成

**中间表示**（Intermediate Representation（IR））
- 在翻译过程中表示源程序的数据结构称为IR
- 例如：抽象语法树（AST）
对中间代码的需要：
- 抽象语法树与目标代码不同，尤其是在控制流构造的表示方面
中间代码：
- 以更类似于目标代码的顺序形式表示语法树

#### 三地址代码（Three-Address Code）
三地址码的最基本指令具有一般格式：`x=y op z`，它表示表达式的计算。
- x，y，z是名称、常量或编译器生成的临时名称
- op代表任何算术或逻辑运算符，例如+、and
- 三地址码来自这种形式的指令：通常x、y和z分别代表内存中的一个地址
![[images/Pasted image 20250625100834.png]]

编程语言中的每种构造的三地址代码：
- 赋值语句的形式为`x=y op z`，其中op为二元运算
- 赋值语句的形式为`x=op y`其中op为一元运算
- Copy语句的形式为`x=y`，其中y的值分配给x
- 无条件跳转`goto L`
- 条件跳转，如`if B goto L`，`if_false B goto L`
- 语句`Label L`表示跳转地址的位置
- `read x`
- `write x`
- 语句`halt`用于标记代码的结束

### 基本代码生成技术（Basic Code Generation Techniques）
语法导向的翻译：
- 作为综合属性的中间代码或目标代码
- 代码生成可视为一种属性计算
- 中间代码可以通过语法树的后序遍历生成
- 中间代码可在解析过程中生成

#### 为表达式/赋值中的数组引用生成TAC
- 如果数组元素存储在连续位置的块中，则可以快速访问数组元素
- 为数组引用生成代码的主要问题是将地址计算关联起来
- **地址计算**（Address-calculation）
	- 基于为数组分配的存储的相对地址（base）、数组的布局和数组元素的宽度

- 数组的布局（Layout）：
	- 行优先（逐行（row-by-row））与列优先（逐列（column-by-column））
	![[images/Pasted image 20250625105215.png]]

- 数组引用的三地址生成
	- 如果base是为数组分配的存储的相对地址，并且每个数组元素的宽度为w且行优先布局，则数组引用的地址计算可以是：
		- `A[i]`的元素从位置`base+i*w`开始
		- `A[n][m]`中的`A[i][j]`的元素可以从`base+(i*m+j)*w`开始

#### 控制语句的翻译
- 用于翻译控制语句的属性文法：
	- 代码生成中使用的函数
		- newlabel()：每次调用时都会返回一个新标签
	- 属性：
		- E.true（E.false）时E为true（false）时控制权流向的标签
		- S.next是一个标签，它附加到第一个三地址指令上，要在S的代码之后执行
		- S.begin是一个标签，附加在为S生成的代码的第一条指令上
