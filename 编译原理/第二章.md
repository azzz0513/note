## 词法分析
![[images/Pasted image 20250619145944.png]]

- **词法单元**（token）由一个词法单元名和一个可选的属性值组成。词法单元名是一个表示某种词法单位的抽象符号，比如一个特定的关键字，或者代表一个标识符的输入字符序列。词法单元名字是由词法分析器处理的输入符号。
- **模式**描述了一个词法单元的词素可能具有的形式。当词法单元是一个关键字时，它的模式就是组成这个关键字的字符序列。对于标识符和其他词法单元，模式是一个更加复杂的结构，它可以和很多符号串匹配。
- **词素**（lexeme）是源程序中的一个字符序列，它和某个词法单元的模式匹配，并被词法分析器识别为该词法单元的一个实例。

- **扫描阶段**主要负责完成一些不需要生成词法单元的简单处理，比如删除注释和将多个连续的空白字符压缩为一个字符。
- **词法分析**是较为复杂的部分，它处理扫描阶段的输出并生成词法单元。
![[images/Pasted image 20250619151531.png]]
词法分析器在编译器中负责读取源程序，因此它还会完成一些识别词素之外的其他任务。任务之一是过滤掉源程序中的注释和空白；另一个任务是将编译器生成错误信息与源程序的位置联系起来。

在很多程序设计语言中，下面的类别覆盖了大部分或所有的词法单元：
- 每个关键字都有一个词法单元。
- 表示运算符的词法单元。
- 一个表示所有标识符的词法单元。
- 一个或多个表示常量的词法单元，比如数字和字面量字符串。
- 每一个标点符号有一个词法单元，比如左右括号、逗号、分号。


### 词素和词法单元
词法单元提供了一种根据词素提供的信息进行分类的方法：
- 一些词法单元可能仅与一个简单的词素有关：
	- 关键字。例如if、while这种
- 一写词法单元与多个不同的词素有关：
	- 所有变量名称、所有可能的数字，所有可能的字符串等

将一组词素与词法单元关联：
- We might associate the “number” token with the set { 0, 1, 2, …, 10, 11, 12, … }
- We might associate the “string” token with the set { "", "a", "b", "c", … }
- Some tokens might be associated with only a single lexeme:
	- We might associate the token for the keyword while with the set { while }.


**字母表**（Alphabet）：一组符号的的集合，如∑={0, 1}，A={a, b, c}
**字符串**（string）：某个字母表上的字符串是从该字母表中提取的有限符号集合
- 0，00，10是∑={0, 1}的字符串
- a，ab，aaca是A={a, b, c}的字符串
- 空字符串ε（长度为0的串）

**语言**（language）是某个给定字母表上的一个任意的可数的串集合。
- {ε}是一个语言
- 空集也是一个符号

### 正则表达式
**正则表达式**（regular expression）是一系列描述，可以用来捕获某些语言。

每个正则表达式r表示了一个语言L(r)，这个语言也是根据r的子表达式所表示的语言递归地定义的。
归纳基础：
- ε是一个正则表达式，L(ε)={ε}，即该语言只包含空串。
- 如果a是∑上的一个符号，那么α是一个正则表达式，并且L(α)={a}。也就是说，这个语言仅包含一个长度为1的符号串a。
归纳步骤：假定r和s都是正则表达式，分别代表语言L(r)和L(s)，那么：
- `(r) | (s)`是一个正则表达式，表示语言`L(r)UL(s)`
- `(r)(s)`是一个正则表达式，表示语言`L(r)L(s)`
- `(r)*`是一个正则表达式，表示语言`(L(r))*`
- `(r)`是一个正则表达式，表示语言`L(r)`

- 一元运算符`*`具有最高的优先级，并且是左结合的
- 连接具有次高的优先级，它也是左结合的
- `|`的优先级最低，并且也是左结合的

可以用一个正则表达式定义的语言成为**正则集合**（regular set）。如果两个正则表达式r和s表示同样的语言，则称r和s**等价**（equivalent），记作r=s。

#### 正则表达式的拓展
- 一个或多个实例。单目后缀运算符`+`表示一个正则表达式及其语言的正闭包。也就是说，如果r是一个正则表达式，那么(r)+就是表示语言(L(r))+。`+`和`*`具有同样的优先级和结合性。
- 零个或一个实例。单目后缀运算符`?`的意思是“零个或一个出现”。也就是说，r?等价于`r|ε`。
- 字符类。一个正则表达式`a1|a2|...|an`可以缩写为`[a1a2...an]`。
	- `[abc]`是`a|b|c`的缩写
	- `[a-z]`是`a|b|...|z`的缩写


### 有限自动机（NFA和DFA）
正则表达式可以使用**有限自动机**（finite automata）来实现。
有限状态机主要由两种：
- NFA（不确定的有限自动机）：对其边上的标号没有任何限制。一个符号标记离开同一状态的多条边，并且空串ε也可以作为标号。
- DFA（确定的有限自动机）：对于每个状态及自动机输入字母表中的每个符号，有且只有一条离开该状态、以该符号为标号的边。
![[images/Pasted image 20250619162827.png]]

#### NFA（Nondeterministic Finite Automata）
一个NFA接受输入字符串x，当且仅当对应的转换图中存在一条从开始状态到某个接受状态的路径，使得该路径中的各条边上的标号组成符号串x。路径中的ε标号将被省略，因为空串不会影响到根据路径构建得到的字符串。

一个NFA由以下部分组成：
- 一个有穷的状态集合S
- 一个输入符号集合∑，即输入字母表（input alphabet）
- 一个转换函数（transition function），它为每个状态和`∑U{ε}`中的每个符号都给出了相应的后继状态（next state）的集合
- S中的一个状态s0被指定为开始状态
- S中的一个子集F被指定为接受状态


- 同一个符号可以标记从同一状态出发到达多个目的状态的多条边
- 一条边的标号不仅可以是输入字母表上的符号，也可以是空符号串ε
![[images/Pasted image 20250619165034.png]]

#### DFA（Deterministic Finite Automata）
- 没有输入ε之上的转换动作
- 对每个状态s和每个输入符号a，有且只有一条标号为a的边离开s

#### 子集构造法

从给定的NFA M构造DFA M’的算法：
- 计算M的起始状态的ε_closure，这将成为M‘的起始状态
- 对于这个集合，以及每个后续集合S，我们计算每个字符a∈∑上的转换Sa，这定义了一个新状态和一个新的Transition。
- 继续此过程，知道不再创建新状态或过渡
- 标记为M的接受状态为接受

### 最小化DFA（minimizing DFA）
最小化一个DFA的状态数。
![[images/Pasted image 20250619212841.png]]
它们都是正则表达式`a*`的DFA，但后者是最小的。
理论：给定任何DFA，都有一个等效的DFA其中包含最小数量的状态，并且此最小状态DFA是唯一的。

等价状态：如果s和t是两种状态，则当且仅当满足以下条件时，它们才等效：
- S和T都是接受状态或不接受状态。
- 对于每个字符a∈∑，s和t在a上具有到等效位置的过渡
![[images/Pasted image 20250619213228.png]]

状态最小化算法（state minimization algorithm）：
将状态集拆分为一些不相交的集合，这样一个集合的状态彼此等效，而不同集合的任意两个状态是可以区分的。
- 首先将状态集分为两个集合，一个里面全为接受状态，另一个全为非接受状态
- 考虑每个子集中每个字符'a'的过渡，确定子集中的所有状态是否等效，还是应该拆分子集
	- 如果一个子集中有两个状态s和t，它们在'a'上具有转换，沿着到达的两个状态中只有一个为接受状态，则说'a'区分状态s和t。如果存在某个可以区分状态s和状态t的串，那么它们就是可区分的（distinguishable）
	- 正在考虑的state集必须根据其‘a’过渡到达的位置进行划分
- 继续这个过程，知道所有集合中都只包含一个元素（原始DFA最小），或者知道集合不再进一步拆分

### 冲突解决（confliction resolutions）
**语义冲突**（Lexing Ambiguities）
![[images/Pasted image 20250619221207.png]]
- Assume all tokens are specified as regular expressions.
- Algorithm: Left-to-right scan.
- Tiebreaking rule one: Maximal munch.
- Always match the longest possible prefix of the remaining text.

给定一组正则表达式，如何使用它们实现maximal munch
- 将表达式转换为NFA
- 并行运行所有所有NFA，跟踪最后一次匹配
- 当所有自动机都卡住时，报告最后一个匹配项并在该点重新开始搜索
